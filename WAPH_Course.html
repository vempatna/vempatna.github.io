Hackathons

  • Hackathon - 1

    Repository Link: Hackathon-1 Repository Link

    Overview of Hackathon - 1

    During hackathon-1, we employed an "echo.php" file as a basic server code with a request function capable of accepting both GET and POST requests. The hackathon comprised seven levels, each addressing and resolving security issues present in the initial server code to eliminate vulnerabilities. In Level 0, we attempted to display an alert using an input form, and in Level 1, we achieved the same result through a GET request by embedding the script directly in the header. Level 2 involved displaying an alert by receiving input through a POST request, utilizing client-side HTML code with a form to gather user input. Starting from Level 3, our focus shifted towards bypassing script tags and preventing the execution of JavaScript code when inputs contained such elements. For Level 5, we implemented code to restrict the input of alerts, requiring the use of the "confirm" method for bypassing. Finally, in Level 7, we developed server-side code to block any HTML code input and execution in the client browser. Throughout this process, we gained a deeper understanding of Cross-Site Scripting (XSS) attacks and SQL code injections in the client. In Task 2, we implemented input validation and sanitation techniques for both the "echo.php" code and the HTML code used in Lab-2. These measures aimed to eliminate security vulnerabilities and enhance the overall efficiency of the code. Our approach involved employing various functions and patterns to validate inputs, ensuring a robust defense against potential attacks and injections.In summary, the hackathon provided valuable insights into code vulnerabilities and equipped us with strategies to mitigate security risks, emphasizing the importance of input validation and sanitation techniques in building secure applications.
